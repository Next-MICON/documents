# 18の計画書とは
正直これを作成するのがPMBOKのメインな気がする<br>
一回作ればチームの資産として生きるので、今回やっとけばあとはテーラリングするだけでいいと思います
## 計画書
- 補助マネジメント計画書
  - [スコープマネジメント計画書](../template/scope_management_plan.md)
    - プロジェクトスコープ記述書をどのような活動で作成するかを書く
      - プロジェクトスコープ記述書とは
        - プロジェクトの作業範囲や成果物、前提条件や制約条件をまとめた文書
      - 例えば
        - 「ステークホルダと会議を行う」など
    - プロジェクトスコープ記述書から、[WBS](./what_is_wbs)をどのような活動で作成するかを書く
    - 作成したプロジェクト記述書と、[WBS](./what_is_wbs.md)を、どのようにして承認するかの手順を書く
    - 完成したプロダクトを、どのように受け入れるのかを書く
      - 正式にプロジェクトを終結させるためにどのような手順をとっていくのか
      - 例えば
        - 「スコープ記述書と照らし合わせて検証する」など
  - [要求事項マネジメント計画書](../template/request_management_plan.md)
    - どのようにしてステークホルダの要求を分析、文書化し、管理していくかを書く(要件定義の方法)
      - どのようにして要求事項を収集していくのか書く
      - どのようにして要求事項を順位付けするのかを書く
      - どのようにして要求事項文書を作成するのかを書く
      - 要求事項文書の変更をどのようにして承認するかを書く
  - [スケジュールマネジメント計画書](../template/schedule_management_plan.md)
    - スケジュールの作成、監視、コントロールに必要な基準と活動を書く
      - プロジェクト・スケジュール・モデルの作成・決定
      - プロジェクト・スケジュール・モデルの維持
      - リリースとイテレーションの期間
      - 測定単位
      - 組織の手続きとの結びつき
      - 正確さのレベル
      - コントロールしきい値
      - パフォーマンス測定値の規則
        - ここまではスクラムについて詳しく書けばよさそうかな？
      - 報告書式
  - [コストマネジメント計画書](../template/cost_management_plan.md)
    - お金以外にも作業量の管理についても書く
      - 測定単位
      - 精密さのレベル
      - 正確さのレベル
      - 組織の手続きとの結びつき
      - コントロールしきい値
      - パフォーマンス測定値の規則
        - ここまではスクラムについて詳しく書けばよさそうかな？(今回お金払わなきゃいけないことはあんまりなさそうだし)
      - 報告書式
      - その他の詳細情報（資金調達方法や為替変動への対応など）
  - 品質マネジメント計画書
    - 品質目標を達成するために適用される方針や手続き、ガイドラインの実行方法を書く
      - プロジェクトが使用する品質基準
        - ISOとかJISとか(まだよくわかってない)
      - プロジェクトの品質目標
        - 「1スプリントでのインクリメントのバグ2個以下」とか
      - 品質に関する役割と責任
      - 品質レビューを受けるプロダクトとプロセス
      - プロジェクトのために計画された品質のコントロールおよび品質のマネジメントの活動
      - プロジェクトで使用する品質ツール
      - 不適合、是正処置の手続き、および継続的改善の手続きなど、プロジェクトに関連のある主要な手続き
    - [眠いのでここ参照](https://ssaits.jp/promapedia/documents/qmp.html)
  - 資源マネジメント計画書
  - コミュニケーションマネジメント計画書
  - リスクマネジメント計画書
  - 調達マネジメント計画書
  - ステークホルダエンゲージメント計画書
  - 変更マネジメント計画書
  - コンフィギュレーションマネジメント計画書
- 追加構成要素
  - スコープベースライン
  - スケジュールベースライン
  - コストベースライン
  - パフォーマンス測定ベースライン
  - プロジェクトライフサイクルの記述
  - 開発アプローチ
