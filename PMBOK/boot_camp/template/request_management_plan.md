# 要求事項マネジメント計画書
- 要求事項の収集
    収集方法を記述
- 要求事項順位付け
  - 順位付けの方法を記述
- 要求事項の定義
  - どのようにして要求事項文書(要件定義書)を作成するかを記述
- WBS作成
  - WBSの作成要領
    - アジャイルだとプロダクトバックログの作成要領
    - プロジェクトスコープ記述書から、WBSを作成する方法を書き出す
  - WBSの維持と承認方法
    - どのようにしてWBSを維持するか
    - どのような手順でWBSの変更を承認するか
  - このへんスコープマネジメント計画書とかぶるので、参照すればいいかな？
- 要求事項検証
  - プロダクトが要求事項に合っているかどのようにして確認するかを記述
- 要求事項のコントロール
  - 差異分析方法
    - プロダクトが要求事項とずれているかどうかを判断する方法
  - 是正処置の提案方法
    - ずれていた場合どのようにして是正処置を提案するか
  - などなど、プロダクトの要求事項を管理する方法を記述
