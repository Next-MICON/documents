# デイリースクラムとは
- [スプリントバックログ](./what_is_sprint_backlog.md)の残り作業を確認
- このまま進めてスプリントゴールが達成できるかを確認
- 15分以内で行う。問題が発覚した場合は、関係者のみデイリースクラムの後に話し合う
  - 起こった問題はすぐに解決する
- 以下の項目を確認
  - スプリントゴール達成のために、自分が昨日やったことは何か
  - スプリントゴール達成のために、自分が今日やることは何か
  - スプリントゴール達成のために、障害となるものはあるか
- どんなに小さな問題でも、必ず共有する
  - 例えば
    - 「思ったより時間がかかってしまった」とか
    - 「ちょっとタスクが漏れていた」とか
  - なんで？
    - 今は小さな問題に見えるものも、後々に大事に発展する可能性がある
- 全員がデイリースクラムの意義をよく理解すること
  - 進捗報告会になってしまっては意味がない
  - 開発チーム内の問題の共有と改善がとても大事
