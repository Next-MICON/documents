# スプリントプランニングとは
- スプリントゴールを設定
  - 今回のスプリントで何をするか(どのプロダクトバックログをするか)
  - ベロシティとキャパシティ(今回のスプリントで書けることのできる時間)を考慮する
  - プロダクトバックログリファインメントを行う
    - 内容
      - 中身を具体的にする
      - メンバー全員の疑問点を解消する
      - 何ができたら完成なのか定義する
      - プロダクトバックログの項目を1日で終わる作業単位に分割する
      - 分割した作業に対して見積もりを行う
    - 時期
      - スプリント開始直前だと間に合わない可能性があるので、余裕を持って行う
    - タイムボックス
      - スプリントの10%以下の時間
